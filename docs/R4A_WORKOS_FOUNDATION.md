# R4A CommonGround WorkOS foundation

Status: acceptance defined before runtime mutation. Verification is `NOT_RUN`.

The private MPES is prime product authority. Workspace
`../docs/OmniOS_20260714.md` is proportional supplementary assurance guidance,
not product authority or a demonstrated runtime. This packet creates the
smallest reversible WorkOS composition layer inside CommonGround.

## Observed baseline

- CommonGround already owns Facilitation and Decision Analysis in one readable
  native-module runtime.
- IndexedDB `commonground-suite` v4 has thirteen app-owned stores. The matter
  registry, workflow routes, revision checks, imports, recovery, worker, cache,
  manifest, and compatibility behavior are authoritative.
- TS-Dash has no recovered readable source. PMQuiz content remains quarantined.
  Knowledge has no accepted owner, schema, storage, recovery, or UI contract.
- CommonGround already consumes design contract `1.1.0` and app-scoped PWA
  assurance `1.0.0`.

## Target composition

| Module | R4A state | Owner | Mutation authority |
| --- | --- | --- | --- |
| Collaboration | active | CommonGround | CommonGround only |
| Decisions | active | CommonGround | CommonGround only |
| Insights | inactive metadata | TS-Dash remains independent | none |
| Learning | inactive metadata | PMQuiz remains independent | none |
| Knowledge | inactive metadata | no accepted runtime owner | none |

Assumptions: CommonGround v4 and its current matter registry remain canonical;
the existing two workflow families are the only active WorkOS capabilities;
inactive entries grant no route, import, storage, worker, or activation.

Guarantees: the shell/catalog are pure versioned metadata; the adapter delegates
to existing CommonGround matter contracts; all mutations remain app-owned;
unknown/inactive modules fail closed; rollback is code-only.

Non-goals: no TS-Dash rewrite/import, PMQuiz migration, Knowledge runtime,
shared store, cross-app read, sync/event bus, universal worker, backend,
account, telemetry, AI, dependency, content change, route retirement, archive
change, or publication.

## Acceptance matrix

1. Versioned app-owned catalog, shell, and adapter expose exactly two active
   modules and three inert entries with explicit prerequisites.
2. Existing matter types, routes, suitability gates, next-step rules, hard
   constraints, revisions, stores, formats, migration, reset, PWA, and recovery
   remain exact.
3. Main navigation is generated by the WorkOS shell; active module boundaries
   are visible and understandable without exposing inactive modules as actions.
4. Deterministic tests prove purity, immutability, ownership, delegation parity,
   schema/PWA invariants, and absence of cross-app runtime capability.
5. Browser and existing suites prove keyboard/pointer/touch operation, focus,
   modal/status behavior, 320–3840 px and 200%-equivalent reflow, forced color,
   reduced motion, file/subpath/offline/update/LKG, malformed/quota/partial/
   stale/multi-tab recovery, and foreign-scope survival.
6. Manual assistive-technology, qualified domain/content review, real browser
   quota/eviction, owner acceptance, deployment, and publication remain
   `NOT_RUN`.

Safe failure: an invalid catalog or missing shell asset rejects the candidate;
the last-known-good CommonGround shell and all user data remain untouched.
Rollback reverts R4A code/styles only.
