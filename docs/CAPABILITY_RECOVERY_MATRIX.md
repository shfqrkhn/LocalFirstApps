## Gate Boundaries

- R3A–R3C add only an app-owned LifeOS shell contract plus Noodle-owned pure Reflection and Flexx-owned pure Strength/controller seams. CommonGround, HealthOS, Noodle, and Flexx retain separate stores, transactions, workers, cache prefixes, formats, routes, and recovery; no data was migrated.
- `npm run test:local` is the deterministic local closure gate. It does not contact Pages.
- `npm run test:live` is a read-only deployment check and remains a separate external gate.
- File-mode tests prove a safe, usable fallback notice and return link; they do not claim that module/worker/PWA runtimes work under `file://`.
- Automated tests use synthetic local data in isolated browser contexts. They never inspect browser profiles, accounts, or personal data.
- Current automation does not certify every screen reader, browser-specific storage eviction policy, or longitudinal multi-device workflow.
- Quota, missing/corrupt assets, and eviction are deterministic simulations. Storage estimates are advisory and may be unavailable; tests do not claim control over a browser's real eviction decisions.
- HealthOS derives elapsed time from persisted instants and exposes clock/timezone anomalies, but operating-system suspension and background audio, vibration, notification, or wake-lock delivery remain outside browser-app control. Manual correction and user review are the recovery path.
- HealthOS is not deployed in this packet. The live gate continues to cover only the six pre-existing public routes and must not be cited as HealthOS deployment evidence.
